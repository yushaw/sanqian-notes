# 本地文件夹笔记本 V1 规划文档

## 1. 文档目标

本文档用于冻结“本地文件夹笔记本（类似 Obsidian Vault）”一期范围，指导产品、设计、前后端和测试协作落地。

- 产品目标：支持将本地目录挂载为笔记本，直接管理 `.md` / `.txt` 文本文件。
- 交互目标：与当前笔记体验融合，尽量复用现有编辑器与搜索能力。
- 实施目标：优先交付用户感知价值高的基础能力，控制一期复杂度。

---

## 2. 背景与设计方向

当前笔记本基于应用内数据模型，用户希望新增“本地文件夹”模式，形成两类数据源并存：

1. 应用内笔记本（现有）
2. 本地文件夹笔记本（新增）

体验方向参考 Apple Notes/Obsidian：

- 左栏继续作为“笔记本入口”。
- 进入笔记本后，以目录树方式组织内容。
- 编辑器能力尽量复用，减少学习成本。

---

## 3. 范围定义

### 3.1 In Scope（V1）

说明：除非特别标注，以下“文件系统规则”仅适用于 `local-folder` 数据源。

1. 左栏“笔记本 +”支持“添加本地文件夹”。
2. 本地文件夹和已有笔记本并存。
3. 本地文件夹目录树最多展示三级（根目录算一级）。
4. local-folder 超过三级的目录/文件不展示、不索引、不参与搜索。
5. 本地文件夹仅支持文件类型：`.md`、`.txt`。
6. 支持创建文件夹、创建文件（新建文件默认 `.md`，新建文件夹无扩展名）。
7. 支持重命名，重名冲突时阻止并提示。
8. 删除本地文件夹笔记本：解除挂载，不删除磁盘文件。
9. 删除本地文件/文件夹：删除磁盘内容，二次确认，移动到系统废纸篓（文件夹递归）。
10. 支持编辑并保存到本地文本文件。
11. 自动保存：延迟自动保存 + 失焦保存 + 切换保存 + `Cmd+S`。
12. 外部改动冲突提示，逻辑参考 VSCode（重载/覆盖/另存）。
13. 纳入全文搜索（按入口决定搜索范围）。
14. 支持多编码文本；无法自动识别时允许用户手动选择编码；保存保持原编码与原换行符。
15. 隐藏项默认不展示（如 `.git`、`.obsidian`、`node_modules`、`.DS_Store`）。
16. 本地大文件处理：`>10MB` 给出友好提示，V1 不支持编辑。
17. 同一路径本地文件夹不允许重复挂载；允许父子目录同时挂载。

### 3.2 Out of Scope（V1 不做）

1. git 同步。
2. 云同步。
3. 插件生态兼容（含 Obsidian 插件/双链高级生态）。
4. 附件管理（图片/PDF/Office）在本功能中扩展支持。
5. 多目录聚合为一个 Vault。
6. 符号链接（symlink/快捷方式）跟随与解析（V1 不处理）。
7. internal 笔记本目录树与 local-folder 完全一致的交互（延后到 V1.5+）。

## 4. 关键产品规则（冻结）

| 主题 | 规则 |
|---|---|
| 层级规则（local-folder） | 根目录为第 1 级，最多显示到第 3 级 |
| 深层内容（local-folder） | 第 4 级及以下全部不展示、不索引 |
| 选中目录展示（local-folder） | 选中某个文件夹时，列表展示该文件夹下全部笔记（含子目录递归，受三级限制） |
| 挂载判重 | 同一 canonical 路径禁止重复挂载；父子路径允许同时挂载 |
| 删除笔记本（local-folder） | 仅解除挂载，不影响磁盘文件 |
| 删除笔记本（internal） | 删除笔记本实体后，笔记统一转为未分类（`notebook_id = NULL`） |
| 删除笔记（local-folder） | 删除磁盘文件，二次确认，移入废纸篓 |
| 删除文件夹（local-folder） | 递归删除，二次确认，移入废纸篓 |
| 跨挂载删除保护 | 若删除目标命中其他已挂载根路径，删除前必须提示受影响挂载并二次确认 |
| 深度创建约束（local-folder） | 在第 3 级目录下禁止再创建子文件夹；允许创建文件 |
| 新建文件（local-folder） | 默认创建 `*.md`，名称走 i18n |
| 搜索范围 | 由入口决定：全局 / 当前笔记本 / 当前文件夹子树 |
| 全局搜索去重 | 父子目录同时挂载时，全局搜索按 canonical 文件路径去重 |
| 去重后排序稳定性 | 全局搜索去重后按统一排序键稳定输出（`score DESC`，同分按 `canonical_path ASC`） |
| 隐藏项（local-folder） | 默认忽略隐藏文件及系统目录，不提供开关 |
| 冲突处理（local-folder） | 参考 VSCode：重载、覆盖、另存 |
| 大文件（local-folder） | 超过 10MB 不支持编辑，提示用户 |
| symlink（local-folder） | 不展示、不跟随、不索引 |
| 写盘安全（local-folder） | 文本保存必须使用原子写入（临时文件 + fsync + rename） |

## 5. 交互流程

### 5.1 添加本地文件夹

1. 用户点击左栏“笔记本 +”。
2. 选择“添加本地文件夹”。
3. 打开系统目录选择器并授权。
4. 对目录做 canonical 路径判重；若已挂载同一路径则阻止并提示。
5. 允许父目录与子目录同时挂载（仅禁止完全相同路径重复挂载）。
6. canonical 生成优先 `realpath + normalize`；若 `realpath` 失败，提示错误并中止挂载。
7. 扫描目录（仅三级、仅 `.md/.txt`、忽略隐藏项）。
8. 扫描完成后，左栏新增该本地笔记本。

### 5.2 浏览与编辑

1. 进入本地文件夹笔记本后，中栏展示目录树（最多三级）。
2. 选中某个文件夹时，笔记列表展示该文件夹及其子目录中的所有笔记（递归，最多三级）。
3. 点击文件打开编辑器。
4. 编辑触发自动保存机制，或用户按 `Cmd+S` 主动保存。
5. 外部改动检测到冲突时，弹出冲突处理选项。

### 5.3 创建与删除

1. 在当前目录新建文件夹/文件（仅新建文件默认 `.md`）。
2. 若当前目录已是第 3 级，禁止创建子文件夹并提示原因。
3. 删除前检测是否影响其他已挂载本地笔记本（父子挂载场景）。
4. 若命中其他挂载，确认弹窗显示受影响挂载列表与数量。
5. 删除文件或文件夹时弹二次确认，执行后移入废纸篓。
6. 删除本地笔记本仅解除挂载。

## 6. 搜索策略

按“入口”确定范围，减少多入口行为不一致：

1. 全局搜索入口：搜索全部笔记本（应用内 + 本地）。
2. 笔记本内搜索入口：搜索当前笔记本。
3. 文件夹内搜索入口：搜索当前文件夹及其子树（三级限制内）。

实现约束：

1. 所有搜索入口必须通过统一 `SearchScopeResolver` 注册 `entryId -> scope` 映射。
2. 未注册入口禁止直接调用搜索服务（防止范围漂移）。
3. `SearchScopeResolver` 变更需配套回归测试。
4. 全局搜索结果按 canonical 文件路径去重，避免父子挂载重复命中。
5. 去重后排序固定为 `score DESC`，同分按 `canonical_path ASC`，避免结果抖动。

V1 基线映射（可扩展）：

| entryId | scope |
|---|---|
| `global_search` | `global` |
| `notebook_search` | `current_notebook` |
| `folder_search` | `current_folder_subtree` |

索引限制：

- local-folder 第四级及以下内容不进入索引。
- local-folder 隐藏项不进入索引。
- local-folder symlink 不进入索引。

## 7. 数据与技术方案

### 7.1 数据源抽象

引入统一数据源接口，隔离“应用内数据”和“本地文件系统”：

- `NotebookSourceType = 'internal' | 'local-folder'`
- `notebooks` 表作为统一笔记本主表，新增 `source_type` 字段（默认 `internal`）。
- 新增 `local_folder_mounts` 表，保存本地挂载扩展信息：
  - `notebook_id`（PK，FK -> `notebooks.id`，`ON DELETE CASCADE`）
  - `root_path`
  - `canonical_root_path`（UNIQUE）
  - `status`（active / permission_required / missing）
  - `created_at` / `updated_at`
- local 文档稳定标识采用 `localDocId = {notebookId}:{normalizedRelativePath}`。
- 挂载判重按 `canonical_root_path` 执行：相同路径拒绝，父子路径允许。
- `canonical_root_path` 由 `realpath + normalize` 生成（在大小写不敏感平台做 case-fold）。
- 若 `realpath` 失败（权限/离线盘/路径不可达），挂载失败并提示用户，不进入扫描。
- `realpath` 失败错误码约定：
  - `LOCAL_MOUNT_PATH_PERMISSION_DENIED`
  - `LOCAL_MOUNT_PATH_UNREACHABLE`
  - `LOCAL_MOUNT_PATH_NOT_FOUND`

说明：internal 笔记本目录树字段（如 `notes.folder_path`）延后到 V1.5+，不纳入 V1 schema。

说明：删除本地笔记本时仅删除挂载记录，不操作磁盘。

### 7.2 目录扫描与监听

1. 首次挂载：全量扫描（深度 <= 3，过滤规则生效）。
2. 日常更新：文件监听（新增/修改/删除/重命名）。
3. 变更后增量更新树与索引。
4. 文件夹选中时，列表通过目录子树聚合笔记（递归到第 3 级）。

建议监听防抖（例如 300ms-800ms）避免频繁抖动。

### 7.3 文件过滤规则

- 允许扩展名：`.md`、`.txt`
- 忽略目录：`.git`、`.obsidian`、`node_modules`
- 忽略文件：`.DS_Store` 及隐藏文件
- 忽略 symlink

### 7.4 编码与换行

1. 打开时自动检测编码。
2. 无法识别时提示用户选择编码并可记忆该文件选择。
3. 保存时保持原编码。
4. 保存时保持原换行符（LF/CRLF）。

### 7.5 保存与冲突

保存触发：

1. 停止输入延迟触发（建议 800ms-1200ms）
2. 编辑器失焦
3. 文件切换
4. 手动 `Cmd+S`

保存可靠性：

1. 所有写盘必须走原子写入（临时文件 -> `fsync` -> `rename`）。
2. 冲突检测采用组合信号：`mtime + size + content hash(可选)`，降低误判。

冲突弹窗选项：

1. 重新加载磁盘版本
2. 覆盖磁盘版本
3. 另存为副本

### 7.6 删除行为

- 文件删除：调用系统废纸篓 API。
- 文件夹删除：递归移动至系统废纸篓。
- 删除前执行“挂载影响分析”，若命中其他挂载根路径需显式告知。
- 二次确认文案需显示对象名称与受影响文件数量。

### 7.7 权限策略

- 使用时主动校验权限。
- 权限丢失时标记笔记本状态并引导用户重新授权。
- 挂载记录保留，避免用户重复配置。

### 8.1 新建文件默认名

- `zh-CN`：`无标题.md`、`无标题 2.md`
- `en-US`：`Untitled.md`、`Untitled 2.md`

### 8.2 新建文件夹默认名

- `zh-CN`：`新建文件夹`、`新建文件夹 2`
- `en-US`：`New Folder`、`New Folder 2`

### 8.3 冲突与删除文案

相关提示均接入 i18n 字典，不在代码内硬编码中文。

## 9. 里程碑与拆分建议

### M1：数据源与挂载基础

1. 新增统一数据源模型（`source_type` + `local_folder_mounts`）
2. “添加本地文件夹”入口与授权
3. 扫描并渲染 local-folder 三级目录树（含目录子树聚合列表）
4. 完成同路径挂载判重（父子目录可共存）

### M2：文件操作闭环

1. 打开、编辑、保存 `.md/.txt`
2. 新建文件/文件夹（含第 3 级目录创建约束）
3. 重命名、删除（废纸篓 + 二次确认）

### M3：稳定性与冲突

1. 自动保存全链路
2. 外部变更监听
3. 冲突检测与处理弹窗
4. 权限失效状态恢复

### M4：搜索与国际化收口

1. 本地内容接入全文搜索
2. 入口范围策略对齐
3. i18n 文案补齐
4. 回归测试与发布检查

## 10. 验收标准（MVP）

### 10.1 功能验收

1. 可成功挂载本地目录并显示在左栏。
2. local-folder 按三级规则正确展示目录树，并支持目录子树聚合列表。
3. `.md/.txt` 可打开、编辑、保存并写回磁盘。
4. 新建文件/文件夹成功，默认命名与 i18n 生效。
5. 删除文件/文件夹进入废纸篓并有二次确认。
6. 删除 local-folder 笔记本仅解除挂载；internal 笔记本保持现有删除语义。
7. 外部改动可被检测并触发冲突处理。
8. 全文搜索能覆盖本地笔记，且范围逻辑正确。
9. 选中目录时，列表展示该目录子树下全部笔记且不越过三级限制。
10. 重复挂载同一路径时被阻止；父子目录可同时挂载。
11. 父子目录同时挂载时，全局搜索不会重复展示同一物理文件。
12. 删除命中其他挂载根路径时，会显示受影响挂载提示并需二次确认。
13. 跨挂载删除确认后，受影响挂载会在一次刷新周期内标记为 `missing` 并提示用户。

### 10.2 约束验收

1. local-folder 第四级及以下内容不展示且不命中搜索。
2. local-folder 隐藏项与 symlink 不展示、不索引。
3. 非 UTF-8 等编码文件可通过“选择编码”正确打开。
4. 保存后编码和换行符不被破坏，且写盘采用原子写入。
5. 大于 10MB 文件给出友好提示且不进入编辑。
6. 同一 canonical 路径不可重复挂载。
7. `realpath` 失败时挂载中止并提示，不进入不确定状态。
8. 全局搜索去重后结果顺序稳定（同一查询多次结果顺序一致）。

## 11. 风险与缓解

1. 文件监听在不同平台稳定性差异。
   - 缓解：监听事件防抖 + 全量兜底重扫入口。
2. 编码识别误判导致乱码。
   - 缓解：提供手动编码选择并记忆。
3. 冲突处理路径复杂导致误覆盖。
   - 缓解：默认先提示，不做静默覆盖。
4. 大目录扫描卡顿。
   - 缓解：异步扫描 + 分批渲染 + 进度反馈。
